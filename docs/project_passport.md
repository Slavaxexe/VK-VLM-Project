# Паспорт проекта

## Название

Дообучение русскоязычной визуально-языковой модели для ответов на вопросы по изображениям.

## Актуальность

Визуально-языковые модели объединяют анализ изображения и естественный язык. Для русскоязычного пользователя важно, чтобы модель не только распознавала объекты, но и корректно понимала вопрос и отвечала по-русски. Полное обучение такой модели требует очень больших вычислительных ресурсов, поэтому в проекте рассматривается практичный подход — дообучение существующей модели на открытых русскоязычных данных VK.

## Цель

Получить дообученную VLM, которая отвечает на вопросы по изображениям на русском языке, и экспериментально оценить изменение качества относительно исходной модели.

## Гипотеза

Если дообучить открытую instruction-tuned VLM на русскоязычных примерах из `deepvk/LLaVA-Instruct-ru` методом QLoRA, то качество на независимых русскоязычных наборах `GQA-ru` и `MMBench-ru` вырастет по сравнению с исходной моделью.

## Задачи

1. Изучить формат, состав и примеры открытых VLM-данных DeepVK.
2. Выбрать компактную открытую базовую VLM, пригодную для обучения на одном доступном GPU.
3. Подготовить часть `LLaVA-Instruct-ru` к формату диалогов выбранной модели.
4. Измерить качество исходной модели на фиксированных подвыборках GQA-ru и MMBench-ru.
5. Выполнить QLoRA-дообучение и сохранить LoRA-адаптер.
6. Повторить оценку в тех же условиях.
7. Сравнить метрики и разобрать характерные примеры ответов модели.
8. Оформить model card, отчёт и воспроизводимые ноутбуки.

## Данные и правила эксперимента

`LLaVA-Instruct-ru` используется только для обучения. `GQA-ru` и `MMBench-ru` зарезервированы только для итоговой оценки: их примеры не включаются в обучающую выборку. Это исключает утечку тестовых данных и делает сравнение метрик корректным.

Для первой итерации обучения используется ограниченная воспроизводимая подвыборка обучающей части с фиксированным seed. После проверки пайплайна объём можно увеличить; в отчёте будут записаны точное число примеров, параметры запуска и версия модели.

## Метод

Планируется применить QLoRA: базовая модель загружается в 4-битном виде, а во время обучения изменяются только небольшие низкоранговые адаптеры. Это снижает требования к видеопамяти и позволяет хранить результат как компактный adapter вместо полной копии модели.

Предварительный кандидат на базовую модель — `Qwen2.5-VL-3B-Instruct`. Окончательный выбор будет подтверждён в ноутбуке после проверки совместимости с GPU и формата данных.

## Метрики и ожидаемый результат

Для GQA-ru будет использована доля точных совпадений ответов после нормализации текста. Для MMBench-ru — accuracy выбранного варианта. Оценка будет выполняться на одних и тех же примерах до и после обучения.

Ожидаемый результат проекта:

- LoRA-адаптер дообученной русскоязычной VLM;
- таблица baseline/fine-tuned с метриками GQA-ru и MMBench-ru;
- качественный анализ как минимум десяти примеров;
- воспроизводимые ноутбуки и подробное описание решения.

## Ограничения

Качество и масштаб эксперимента ограничены доступным GPU, поэтому обучение не претендует на создание фундаментальной модели с нуля. Оценка на ограниченных подвыборках является быстрой исследовательской проверкой; при наличии ресурсов её можно расширить до полных тестовых наборов.
