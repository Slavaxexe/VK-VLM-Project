# Model card: F3-10k Qwen2.5-VL-3B QLoRA

## Краткое описание

F3-10k — LoRA-адаптер для `Qwen/Qwen2.5-VL-3B-Instruct`, дообученный на русскоязычных визуальных диалогах из открытого набора `deepvk/LLaVA-Instruct-ru`. Модель получает изображение и вопрос на русском языке и формирует ответ с учётом визуального контекста.

Адаптер создан в рамках учебного проекта VK-VLM-Project. Он предназначен прежде всего для экспериментов с русскоязычными развёрнутыми ответами и объяснениями по изображениям. Это не самостоятельная полная модель: для запуска требуется исходная `Qwen2.5-VL-3B-Instruct`.

## Основные сведения

## Назначение

Подходящие сценарии:

- русскоязычные вопросы по изображениям;
- описание визуальной сцены;
- объяснение наблюдаемых действий и отношений;
- учебные эксперименты с VLM и QLoRA;
- сравнительное исследование parameter-efficient fine-tuning.

Модель не предназначена для принятия решений с высокой ценой ошибки, медицинской или юридической интерпретации изображений, биометрической идентификации и автоматической модерации без проверки человеком.

## Данные

Для обучения использованы 10 000 примеров из обучающей части [deepvk/LLaVA-Instruct-ru](https://huggingface.co/datasets/deepvk/LLaVA-Instruct-ru). Набор содержит русскоязычные диалоги по изображениям и задания типа `complex_reasoning`. Связанные изображения загружались из открытого COCO-архива.

Для согласованной оценки сформирован отдельный holdout из 100 примеров: 50 `conversation` и 50 `complex_reasoning`. Номера строк holdout не пересекаются с исходной 12-тысячной кандидатной выборкой, из которой формировались обучающие подвыборки F1, F2, F3 и A1. GQA-ru и MMBench-ru не использовались в обучении.

## Процедура обучения

Базовая модель загружалась в четырёхбитном формате:

- quantization: NF4;
- double quantization: включена;
- compute dtype: FP16;
- optimizer: `paged_adamw_8bit`;
- gradient checkpointing: включён;
- batch size: 1;
- gradient accumulation: 8;
- эффективный batch size: 8;
- learning rate: `1e-4`;
- scheduler: cosine;
- warmup steps: 38;
- оптимизационных шагов: 1 250;
- максимальная длина последовательности: 1 024;
- диапазон визуальных токенов: от `256 × 28 × 28` до `512 × 28 × 28` пикселей в настройках processor.

Параметры LoRA:

- rank: 16;
- alpha: 32;
- dropout: 0,05;
- target modules: `q_proj`, `k_proj`, `v_proj`, `o_proj`, `gate_proj`, `up_proj`, `down_proj`.

## Динамика обучения

Обучение завершило 1 250 из 1 250 плановых шагов. Средний loss по первым пяти записям равен 1,067, по последним пяти — 0,911; минимальный записанный loss — 0,829. 95-й перцентиль gradient norm составил 3,892. В истории нет ошибок, NaN или признаков расходимости.

Подробные графики находятся в `notebooks/10_training_dynamics_and_summary.ipynb` и `results/figures/10_*.png`.

## Оценка

### Внешние короткие бенчмарки

| Метрика | B0 | F3-10k |
| --- | ---: | ---: |
| GQA-ru Exact Match | 43% | 42% |
| MMBench-ru Accuracy | 74% | 73% |
| GQA-ru BERTScore F1 | 87,47% | 83,77% |

F3 не улучшила строгие короткие ответы и выбор варианта. LLaVA instruction-tuning специализировал модель на более развёрнутом формате, что ограничило перенос на задачи, где ожидается одно слово или буква.

### LLaVA holdout

| Метрика | B0 | F3-10k | Изменение |
| --- | ---: | ---: | ---: |
| BERTScore F1 | 91,15% | 92,31% | +1,16 п.п. |
| Unicode ROUGE-L | 25,83% | 32,53% | +6,70 п.п. |
| Средняя длина ответа | 32,20 | 33,46 | +1,26 слова |

Парный bootstrap на 20 000 итерациях:

- BERTScore: 95% CI изменения `[+0,86; +1,45]` п.п.;
- ROUGE-L: 95% CI изменения `[+3,63; +10,00]` п.п.

Оба интервала не пересекают ноль, поэтому прирост устойчив на выбранном holdout.

## Ограничения

- Оценка выполнена на фиксированных подвыборках по 100 примеров, а не на полном объёме бенчмарков.
- BERTScore и ROUGE-L измеряют сходство с одним референсом и не гарантируют фактическую истинность каждого утверждения.
- Instruction-tuning улучшил согласованный диалоговый формат, но не дал универсального роста на всех типах задач.
- Модель может добавлять правдоподобные, но отсутствующие на изображении детали.
- Данные LLaVA и COCO могут содержать исходные смещения по объектам, ситуациям и стилю описаний.
- Адаптер тестировался на конкретных версиях библиотек и одной потребительской GPU.

## Рекомендации по использованию

- Для короткого ответа явно указывать требуемый формат в prompt.
- При значимой цене ошибки проверять ответ человеком.
- Не интерпретировать BERTScore как вероятность правильности.
- При сравнении с другими моделями использовать одинаковые изображения, prompts и параметры генерации.

## Запуск

Для использования требуется базовая модель и PEFT-адаптер:

```python
from peft import PeftModel
from transformers import AutoProcessor, Qwen2_5_VLForConditionalGeneration

base_id = "Qwen/Qwen2.5-VL-3B-Instruct"
adapter_path = "models/f3_10k_qwen_r16_adapter"

processor = AutoProcessor.from_pretrained(adapter_path)
model = Qwen2_5_VLForConditionalGeneration.from_pretrained(base_id, device_map="auto")
model = PeftModel.from_pretrained(model, adapter_path).eval()
```

Полный код подготовки изображения и генерации приведён в `notebooks/09_llava_holdout_evaluation.ipynb`.
